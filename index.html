Panel de Administración

Gestiona tus productos digitales y configura pasarelas de pago.

Configura tus pasarelas de pago pegando las credenciales de cada servicio. Las claves se guardan en tu navegador (localStorage) y se enviarán al backend cuando proceses pagos reales.

💳

Stripe

Tarjetas de crédito/débito internacionales
No configurado
Acepta Visa, Mastercard, Amex, Apple Pay y Google Pay. Comisión: ~2.9% + $0.30/tx.
Obtén tus claves en dashboard.stripe.com/apikeys
Esta clave se usa en el frontend para inicializar Stripe.js. Es segura para el cliente.
⚠️ Esta clave es privada. En producción NUNCA se expone en el frontend. Se usa solo en tu backend (Node.js/PHP).
Opcional. Se usa para verificar webhooks de Stripe en tu backend.
🅿️

PayPal

Pagos internacionales con saldo PayPal y tarjetas
No configurado
Acepta saldo PayPal, Visa, Mastercard, débito directo. Comisión: ~3.49% + $0.49/tx.
Obtén tus credenciales en developer.paypal.com
Se obtiene creando una app REST en PayPal Developer Dashboard. Usa el Client ID de "Live" para producción o "Sandbox" para pruebas.
⚠️ Privado. Se usa en el backend para capturar pagos y verificar webhooks.
📱

Yappy (Banco General)

Pagos locales Panamá — Banco General
No configurado
Pagos móviles para clientes en Panamá con cuenta Banco General. Comisión: ~1% + ITBMS.
Activa Yappy Comercial desde tu Banca en Línea o contacta botondepagoyappy@bgeneral.com
Se genera desde el panel de Yappy Comercial en Banca en Línea de Banco General.
⚠️ Privado. Se usa en tu backend para autenticar solicitudes de pago con la API de Yappy.
Endpoint GET en tu backend donde Yappy enviará la confirmación de pago.
🖥️

Configuración del Backend

Endpoints necesarios para procesar pagos reales
Para procesar pagos reales, necesitas un servidor backend (Node.js, PHP, Python, etc.) con los siguientes endpoints:
// ═══ STRIPE ═══
POST /api/stripe/create-checkout-session
POST /api/stripe/webhook
// ═══ PAYPAL ═══
POST /api/paypal/create-order
POST /api/paypal/capture-order
// ═══ YAPPY ═══
POST /api/yappy/create-payment
GET  /api/yappy/confirm (IPN callback)
La URL base de tu servidor backend. Los endpoints de pago se construirán sobre esta URL.
